Token input streams and tools converting to and from them..

“What’s up?” “I don’t know,” said Marvin, “I’ve never been there.”

Input is the primary trait used to feed input data into a chumsky parser. You can create them in a number of ways: from strings, slices, arrays, etc.

Structs

  • Emitter
    Struct used in Parser::validate to collect user-emitted errors
  • InputRef
    Internal type representing an input as well as all the necessary context for parsing.
  • Marker
    Represents a location in an input that can be rewound to.
  • Offset
    Represents a location in an input.
  • SpannedInput
    A wrapper around an input that splits an input into spans and tokens. See Input::spanned.
  • Stream
    An input that dynamically pulls tokens from an Iterator.
  • WithContext
    An input wrapper contains a user-defined context in its span, in addition to the span of the wrapped input. See Input::with_context.

Traits

  • BorrowInput
    Implemented by inputs that can have tokens borrowed from them.
  • ExactSizeInput
    Implement by inputs that have a known size (including spans)
  • Input
    A trait for types that represents a stream of input tokens. Unlike Iterator, this type supports backtracking and a few other features required by the crate.
  • SliceInput
    Implemented by inputs that represent slice-like streams of input tokens.
  • StrInput
    A trait for types that represent string-like streams of input tokens
  • ValueInput
    Implemented by inputs that can have tokens borrowed from them.

Type Definitions

    A stream containing a boxed exact-sized iterator. See Stream::exact_size_boxed.
  • BoxedStream
    A stream containing a boxed iterator. See Stream::boxed.
